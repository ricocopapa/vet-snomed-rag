## §2. 폐기 근거 (3 줄 요약)

1. **MRR@10 임계 미달.** M2 0.882 / M3 0.877 vs 임계 0.929 (baseline 0.885 × 1.05). 두 후보 모두 baseline에 미세 열세.
2. **분포 강건성 baseline 우위.** 1,000-pool 한정 setup에서 T7 "feline diabetes" baseline rank 2 vs 후보 rank 5/6. 작은 candidate pool 분별력에서 baseline이 가장 안정.
3. **alpha transfer 가설(SapBERT entity-linking 우월) reject.** 본 setup에서 SapBERT의 UMLS pretraining 강점이 발현되지 않음. 짧은 영어 쿼리 + preferred_term 표면 일치 다수의 본 dataset에서는 baseline MiniLM도 충분.

## §4. 본 결정의 한계 (재발 가능성)

R-8 폐기는 "도메인 특화 임베더 자체의 reject"가 아니라 "본 setup에서 도메인 강점 발현 부재"의 결론이다. 다음 조건이 충족되면 재검토 가치:

- **cross-walk / 약어 / spelling variant 쿼리가 dataset 다수**가 되는 환경 (현재 본 dataset은 < 5%)
- **Pool 크기가 100K+** (1,000은 너무 작아 모든 모델이 perfect에 수렴)
- **다국어 쿼리 비중 ≥ 30%** — 영문 임베더 한계가 결정적 제약이 되는 시나리오

본 산출물(scripts + dataset + 임시 ChromaDB)은 R-9 다국어 임베더 후보 검증에 그대로 재활용 가능. 폐기는 R-8 cycle 종결을 의미하며, 산출물 archive는 추후 결정.
